## 1. 현재 구현 기능 전체 정리

### 1-1. 인증 & 사용자

| 기능                 | 상태    | 설명                                                    |
| -------------------- | ------- | ------------------------------------------------------- |
| 회원가입             | ✅ 구현 | 이메일·비밀번호·이름 입력, 이메일 중복 확인             |
| 로그인               | ✅ 구현 | JWT AccessToken(메모리) + RefreshToken(HttpOnly Cookie) |

| 프로필 이미지 업로드 | ✅ 구현 | 파일 업로드 후 USER 타입으로 서버 저장                  |

## 2. 유저 메인 서비스 FLOW (기존)

1. **유저 생성**
   - 유저는 이메일(id)로 구분
   - 이름, 프로필 이미지 입력 저장
     - 이름은 빈 값 불가
2. **팔로우 기능**
   - 다른 사용자 팔로우 / 언팔로우 가능
   - 자기 자신 팔로우 불가
3. **프로필 노출**
   - 이름, 프로필 이미지, 팔로잉 수, 팔로워 수 표시
   - 팔로잉·팔로워 수 클릭 시 각 유저 목록(이름·프로필) 노출

---

## 3. 피드 서비스 메인 FLOW (기존)

1. **게시물 작성**
   - Tiptap 리치 에디터로 텍스트 입력
   - 공개 범위 선택 (모두 공개 / 팔로워 전용 / 비공개)
   - 게시물 제출
     - 첫 번째 블록은 반드시 H1(제목)으로 시작
2. **댓글 작성**
   - 텍스트 입력 후 제출
   - 댓글 글자수 100자 이하
3. **게시물 상호작용**
   - 좋아요 / 취소 (본인 글 불가)
   - 댓글 좋아요
   - 글·댓글 수정 (본인만, 수정 시간 저장)
   - 글·댓글 삭제 (본인만)
4. **피드 보기**
   - 팔로잉 + 공개 게시글 혼합, 최신순
   - 무한 스크롤
   - 표시 정보: 제목·내용·작성자·프로필·좋아요 수·댓글 수·조회수·수정 여부·발행일·나의 좋아요 여부
5. **댓글 리스트**
   - 작성자 이름·프로필·내용·좋아요 수·수정 여부·발행일·나의 좋아요 여부

## 4. AI 기능 (기존)

1. **RAG 챗봇** (`/ai/chat/stream`)
   - 사용자가 질문 입력 → SSE 스트리밍으로 답변 수신
   - pgvector 기반 유사 게시글 검색 후 GPT-4o-mini로 답변 생성
   - 답변 하단에 출처 게시글 링크 표시
   - 할루시네이션 방지: 유사도 임계값(0.3), 검색 결과 0건 시 LLM 미호출
2. **이미지 AI 분석** (`/ai/image/analysis`)
   - 이미지 업로드 → AI 키워드 목록 반환 → 해시태그 자동 제안
3. **PDF AI 임포트** (`/ai/pdf/import`)
   - PDF 업로드 → AI가 내용 파싱 → 게시글 자동 초안 생성

## 5. 실시간 알림 (기존)

- **LIKE**: 내 게시글에 좋아요 발생 시
- **COMMENT**: 내 게시글에 댓글 발생 시
- **FOLLOW**: 누군가 나를 팔로우 시
- 상단 벨 아이콘 미읽음 배지 실시간 갱신 (SSE)
- 알림 클릭 → 관련 게시글 / 프로필로 이동

## 6. 크리에이터 대시보드 — 신규 기획

> **Why**: 스터디 인증 게시글을 꾸준히 작성하는 사용자가
> 단순 "피드 작성자"를 넘어 **콘텐츠 크리에이터**로 성장할 수 있는
> 데이터 기반 환경을 제공한다.

### 6-1. 진입점 및 UI 구조

```
Navigation Bar
└── Me (프로필 아이콘)
    └── ProfilePage (/profile)
        ├── [기존] 내 정보 탭
        ├── [기존] 좋아요한 글 탭
        ├── [기존] 내가 쓴 글 탭
        └── [신규] 📊 크리에이터 대시보드 탭  ← 추가
```

**라우팅**: `/profile?tab=dashboard` 또는 `/dashboard` 전용 페이지 (추후 확장)

---

### 6-2. 크리에이터 레벨 시스템

작성자의 활동량에 따라 레벨을 부여하여 성장 동기를 제공한다.

| 레벨 | 타이틀               | 조건                               |
| ---- | -------------------- | ---------------------------------- |
| Lv.1 | 🌱 새싹 크리에이터   | 게시글 1개 이상                    |
| Lv.2 | 📖 스터디 메이트     | 게시글 5개 + 팔로워 10명 이상      |
| Lv.3 | ✨ 인플루언서        | 게시글 20개 + 팔로워 50명 이상     |
| Lv.4 | 🔥 스터디 리더       | 게시글 50개 + 팔로워 200명 이상    |
| Lv.5 | 👑 레전드 크리에이터 | 게시글 100개 + 팔로워 1,000명 이상 |

> 레벨은 내 프로필 카드 상단에 배지로 노출되며,
> 다른 사용자 프로필에서도 배지가 표시된다.

#### B. 콘텐츠 성과 분석

**최근 7일 / 30일 / 전체 기간 탭 전환**

| 지표           | 설명                       | 시각화    |
| -------------- | -------------------------- | --------- |
| 조회수 추이    | 기간별 일간 조회수 합계    | 라인 차트 |
| 좋아요 추이    | 기간별 일간 좋아요 수 합계 | 바 차트   |
| 게시글 발행 수 | 기간별 작성 게시글 수      | 바 차트   |
| 팔로워 증감    | 기간별 신규 팔로워 수      | 라인 차트 |

> **Next.js 구현**: `recharts` 또는 `chart.js` 라이브러리 사용
> Purple-Pink 그라디언트 테마 적용 (기존 디자인 시스템 일관성)

**신규 API**:

```
GET /users/me/analytics?period=7d|30d|all
Response: {
  dailyViews: [{ date, count }],
  dailyLikes: [{ date, count }],
  dailyPosts: [{ date, count }],
  dailyFollowers: [{ date, count }]
}
```

#### F. 활동 스트릭 (연속 작성 달력)

### 6-4. 게시글 상세 분석 (포스트 디테일 페이지 확장)

본인 게시글 상세 페이지(`/post/[id]`)에 **작성자 전용 분석 패널** 추가.

```
[작성자에게만 보이는 영역]
┌────────────────────────────────────────────────────┐
│ 📊 이 게시글 성과                                   │
│  👁 조회수 342    ❤️ 좋아요 41    💬 댓글 12       │
│  팔로워에게 노출  182명  /  비팔로워 160명          │
└────────────────────────────────────────────────────┘
```

### 6-5. 크리에이터 공개 프로필 배지

다른 사용자의 프로필 페이지에서 크리에이터 레벨 배지를 표시한다.

```
[홍길동 프로필]
  [프로필 이미지]
  홍길동
  ✨ 인플루언서  ← 레벨 배지 (클릭 시 레벨 설명 툴팁)
  팔로워 87  팔로잉 34
```

### 6-6. 신규 API 목록 요약

| Method | Endpoint                              | 설명                                       |
| ------ | ------------------------------------- | ------------------------------------------ |
| GET    | `/users/me/stats`                     | 총 좋아요·조회수·댓글 수 집계              |
| GET    | `/users/me/analytics?period=`         | 기간별 일간 지표 (조회·좋아요·게시·팔로워) |
| GET    | `/users/me/posts/top?limit=&orderBy=` | 내 인기 게시글 TOP N                       |
| GET    | `/users/me/followers/recent?limit=`   | 최근 팔로워 목록                           |
| GET    | `/users/me/followers/not-following`   | 맞팔 안 된 팔로워 목록                     |
| GET    | `/users/me/hashtag-analytics`         | 해시태그별 성과                            |
| GET    | `/users/me/streak`                    | 스트릭 + 활동 캘린더                       |
| GET    | `/users/{email}/level`                | 공개 크리에이터 레벨 조회                  |

### 6-8. UI/UX 원칙 (Next.js + 기존 디자인 시스템 일관성)

| 원칙                        | 적용 방식                                                     |
| --------------------------- | ------------------------------------------------------------- |
| **Purple-Pink 테마 일관성** | 차트 색상: purple-600 / pink-500 그라디언트                   |
| **모바일 퍼스트**           | 수평 스크롤 카드, safe-area 대응, 터치 최적화                 |

| **AuthGuard**               | 대시보드 전체 페이지에 `<AuthGuard>` 래핑                     |
| **SSR / CSR 분리**          | 정적 레이아웃은 서버 컴포넌트, 차트·상호작용은 `"use client"` |
